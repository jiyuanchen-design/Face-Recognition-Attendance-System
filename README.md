# Face-Recognition-Attendance-System

人脸签到系统项目文档
一、项目概述
本项目是一款基于人脸识别技术的简易签到系统，旨在实现便捷、高效的人员签到与签退管理，替代传统手动签到模式，适用于课堂、小型会议、社团活动等场景。系统具备人脸采集、人员注册、签到签退、记录查询、成员管理等核心功能，操作简单、界面直观，可快速完成人员身份验证与签到状态统计，满足日常小型场景的签到管理需求。
二、技术栈选型
本项目采用Python作为核心开发语言，结合开源库与工具，实现人脸识别、图形界面、数据持久化等功能，技术栈选型兼顾易用性、轻量性和可扩展性，具体如下：
2.1 核心开发语言
Python 3.8+：语法简洁、生态丰富，拥有大量开源视觉处理、图形界面开发库，降低开发难度，提升开发效率，适合快速实现小型应用开发。
2.2 人脸识别相关库
OpenCV：开源计算机视觉库，用于人脸检测、图像预处理（如灰度转换、人脸区域截取），是实现人脸识别功能的基础，提供高效的图像操作接口。

PyTorch：深度学习框架，用于加载预训练的ResNet18模型，提取人脸特征向量，实现人脸相似度匹配，确保身份识别的准确性。

TorchVision：PyTorch配套视觉库，提供预训练模型（ResNet18）、图像变换工具，简化人脸特征提取的开发流程。
2.3 图形用户界面（GUI）开发
Tkinter：Python内置的GUI开发库，轻量、无需额外安装，用于构建系统的可视化界面，包括采集、签到签退、记录查询、成员管理等功能模块，实现用户与系统的交互。
2.4 数据持久化
SQLite3：轻量级嵌入式数据库，无需独立部署，用于存储学生信息（学号、姓名、人脸特征向量）和签到记录（学号、姓名、签到时间、签到状态），操作简单、占用资源少，适合小型项目的数据存储需求。
2.5 其他辅助库
PIL/Pillow：用于图像格式转换，将OpenCV处理后的图像转换为Tkinter可显示的格式，解决图形界面中图像显示的兼容性问题。

NumPy：用于数值计算，处理人脸特征向量的存储、相似度计算（点积运算），提升特征匹配的效率。

datetime：用于获取系统当前时间，记录签到、签退的具体时间，实现时间相关的逻辑判断。
三、项目核心功能实现
系统采用模块化开发，分为人脸采集与注册模块、签到签退模块、记录查询模块、成员管理模块，各模块相互独立又协同工作，具体实现功能如下：
3.1 人脸采集与注册模块
该模块实现人脸图像的采集、特征提取与人员信息注册，是系统正常运行的基础，具体流程如下：
用户输入学号和姓名，点击“采集”按钮，系统调用摄像头，捕获20张人脸图像（确保特征提取的准确性），对图像进行预处理（灰度转换、人脸区域截取）。

通过ResNet18模型提取每张人脸的特征向量，计算20张图像特征向量的平均值，作为该用户的最终人脸特征。

点击“注册”按钮，将用户的学号、姓名、人脸特征向量存储到SQLite3数据库中，完成人员注册；若学号已存在，系统提示“学号已存在”，避免重复注册。
3.2 签到签退模块
该模块是系统的核心功能，实现基于人脸识别的自动签到与签退，简化签到流程，具体实现如下：
签到功能：用户点击“签到”按钮，系统调用摄像头，检测人脸并提取特征向量，与数据库中已注册人员的特征向量进行相似度匹配（相似度阈值设为0.6，确保识别准确率）。

匹配成功后，系统自动记录当前时间，签到状态统一设为“签到完成”（无迟到、缺勤状态），并将签到记录存入数据库，同时在界面提示签到成功。

签退功能：用户点击“签退”按钮，系统同样进行人脸匹配，确认用户已签到后，记录当前时间，签退状态设为“签退完成”，存入数据库，并将用户从“已签到”集合中移除。

异常处理：未检测到人脸时提示“未检测到人脸”；人脸匹配失败时提示“未知人员”；未签到用户尝试签退时提示“未签到”。
3.3 签到记录查询模块
该模块用于查询当日所有签到、签退记录，方便管理人员统计出勤情况，具体实现：
系统默认查询当日（当前日期）的所有签到记录，包括学号、姓名、签到/签退时间、状态（签到完成/签退完成）。

用户点击“刷新”按钮，可实时更新记录列表；点击“重置签到”按钮，可清空当日所有签到记录（需确认操作，避免误删）。

记录列表采用表格形式展示，清晰直观，便于查看和统计。
3.4 成员管理模块
该模块用于管理已注册的人员信息，实现人员的删除、批量清空等操作，具体实现：
用户点击“刷新成员”按钮，可查看所有已注册人员的学号、姓名及当日签到状态（未签到/签到完成，签退成功后自动变为“未签到”）。

选中某一成员后，点击“删除选中”按钮，可删除该成员的所有信息（包括注册信息和签到记录），删除前需确认操作。

点击“清空所有成员”按钮，可删除所有已注册人员的信息（需确认操作，不可恢复），适用于场景重置需求。
四、系统运行流程
启动系统：运行main.py文件，系统初始化数据库、加载已注册人员的人脸特征，启动图形界面。

人员注册：进入“采集”标签页，输入学号、姓名，点击“采集”完成人脸采集，再点击“注册”完成人员添加。

签到操作：进入“签到/签退”标签页，点击“签到”，摄像头启动，完成人脸匹配后，提示签到成功，记录自动存入数据库。

签退操作：进入“签到/签退”标签页，点击“签退”，完成人脸匹配后，提示签退成功，成员状态变为“未签到”。

记录查询：进入“记录”标签页，点击“刷新”，查看当日所有签到、签退记录；必要时可点击“重置签到”清空记录。

成员管理：进入“成员管理”标签页，点击“刷新成员”查看所有成员；可删除选中成员或清空所有成员。
五、系统特点
操作简单：图形界面直观，步骤清晰，无需专业知识，普通用户可快速上手。

识别高效：基于ResNet18模型的人脸特征提取，结合合适的相似度阈值，识别准确率高、速度快。

轻量便捷：采用SQLite3嵌入式数据库，无需独立部署，可直接运行，占用系统资源少。

功能实用：覆盖采集、注册、签到、签退、记录查询、成员管理全流程，满足小型场景的签到管理需求。

稳定可靠：具备完善的异常处理（重复注册、未签到签退、人脸匹配失败等），确保系统稳定运行。
六、总结
本项目基于Python语言，结合OpenCV、PyTorch、Tkinter等技术，实现了一款功能完整、操作便捷的人脸签到系统。系统核心解决了传统签到模式效率低、易出错的问题，通过人脸识别技术实现自动签到签退，同时提供完善的记录查询和成员管理功能，适用于课堂、小型会议等场景。
项目采用模块化开发思路，各模块职责清晰、易于维护和扩展，后续可根据需求优化功能，如增加历史记录查询、导出签到报表、提升人脸识别准确率等，进一步提升系统的实用性和扩展性。
